# Sakuya Translator

面向 MTTool 导出文本的桌面 AI 翻译工具，支持 JSON、CSV、SRT、TXT 的批量翻译、断点续翻和格式保留。

程序会尽量保留原文件格式。翻译完成后，会在 `output` 文件夹里生成新的译文文件。

## 快速开始

下载 Windows x64 解压即用版，解压后运行：

```text
SakuyaTranslator.exe
```

解压即用版已包含 .NET 运行时。运行时请保留发布目录内的 DLL、`Locales` 等文件。

## 第一次使用

1. 打开左侧的“API 配置”。
2. 选择你要使用的平台，比如 DeepSeek、Gemini、OpenAI、OpenRouter 等。
3. 填入 API Key。
4. 填入或选择模型名。
5. 点击保存。
6. 回到底部状态栏，确认当前平台和模型已经选好。

如果你保存了多个平台或多个模型，可以在窗口底部直接切换。

## 开始翻译

1. 打开左侧的“JSON 目录”。
2. 点击添加文件，选择 MTTool 导出的 JSON，或者 CSV、SRT、TXT 文件。
3. 也可以添加整个文件夹，程序会自动加入支持的文件。
4. 点击开始翻译。

多个文件会进入队列。  
如果你中途关闭程序，下次打开时队列还会保留。

## 翻译结果在哪里

翻译完成后，文件会自动保存到程序目录下的：

```text
output
```

文件名会自动带上目标语言后缀，例如：

```text
example-zh.json
example-zh.srt
```

也可以在任务详情里手动点击“导出”。

## 支持的文件

- MTTool 导出的 JSON
- CSV
- SRT 字幕
- TXT 文本

不同文件格式的结构不一样，建议翻译完成后简单检查一下导出结果。

## 支持的 API

目前内置了一些常见平台：

- OpenAI
- Gemini
- DeepSeek
- 小米 MiMo
- OpenRouter
- Kimi
- 通义千问

只要接口兼容 OpenAI 的 `/chat/completions` 格式，一般都可以尝试使用“自定义接口”接入。

## 翻译速度

设置里可以选择翻译速度：

- 保守：最稳，适合额度低或容易限速的 API。
- 普通：推荐默认选择。
- 快速：速度更快，但更容易遇到限速。
- 批量并发：适合额度比较高的 API。
- 自定义：可以手动调整并发、每批数量、间隔等参数。

如果翻译时报错、限速或 API 不稳定，可以把速度调低。

## 断点续翻

程序会保存任务进度。  
如果翻译中途关闭、暂停、失败，下次打开后可以继续。

相关进度文件会保存在：

```text
Projects
```

一般不需要手动处理这些文件。

## 数据和隐私

程序不使用本地数据库。  
配置和运行数据会保存在程序目录附近：

- `Data`：设置、API 配置、队列记录。
- `Projects`：断点续翻进度。
- `Backups`：备份。
- `Logs`：日志。
- `output`：导出结果。

API Key 保存在本地配置文件里。  
不要把已经填好自己 API Key 的程序目录分享给其他人。

## 实验功能

“校对”目前是实验功能，默认关闭。  
如果想测试，可以在设置里的“实验功能”中打开。

这个功能现在还可能误报，建议只当辅助参考，不要完全依赖它判断译文质量。

## 常见问题

### 打不开程序

请确认下载的是 Windows x64 解压即用版，并且没有只单独移动 `SakuyaTranslator.exe`。

### 翻译很慢

可以在设置里选择“快速”或“批量并发”。  
如果 API 报限速，就调回“普通”或“保守”。

### 译文里出现奇怪内容

AI 有时会输出解释、拒绝回复或不符合预期的内容。  
程序会尽量处理这类情况，但还是建议翻译完成后检查关键文本。

### 导出的文件没有看到

先看程序目录下的 `output` 文件夹。  
如果仍然没有，可以进入任务详情手动点“导出”。
